[도입이유]
- 인터페이스를 **직접 구현하는 방식은 코드의 재사용성이 떨어짐**
- **인터페이스의 기능 요구사항 변경 시 해당 인터페이스를 구현한 모든 서브클래스의 코드 수정이 불가피함**

[설계]
- 변경가능성이 있는 행동(역할 혹은 기능)들은 인터페이스로 분리
- 요구사항에 맞게 해당 인터페이스 구현
- 구현체가 아닌 인터페이스 (상위형식)을 참조하도록 구현
- setter와 생성자를 활용하여 **행동을 동적 할당**
